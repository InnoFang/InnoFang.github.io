[翻译] An Algorithm for Subgraph Isomorphism
2019-04-23|算法

[翻译] An Algorithm foir Subgraph Isomorphism

作者：J. R. Ullmann

翻译：Inno Fang

链接：https://www.cs.bgu.ac.il/~dinitz/Course/SS-12/Ullman_Algorithm.pdf

概要 子图同构可以通过蛮力树搜索枚举过程的方式来确定。本论文提出了一种在树搜索中通过推理消除后继结点的方式来提高搜索效率的新算法。为了评估新算法实际用时，对随机图和多种非随机图进行了子图同构（subgraph isomorphism)，分团检测（clique detection），图同构（graph isomorphism）和有向图同构（directed graph isomorphism）实验。

本文还描述算法的一个重要部分的并行异步逻辑内存实现，尽管该硬件实际上尚未构建。硬件实现将允许非常快速地确定同构。

1. 介绍

Corneil 和 Cotlieb[4] 提到，子图同构的一种可能的应用是给定结构式的情况下，找出给定的化合物是否是另一种特定化合物的亚化合物。子图同构可能在场景分析 [1, 10] 中用来检测嵌入在场景中的关系描述对象。类似于子图同构的问题也出现在扭曲形状的识别的研究中，其中任何允许的变形都在限制内保持位置关系。在发现两个图是否是通过 1 : 1 对应关系来保持一致性和发现到两个模式是否通过在已知限制内保持空间关系的形变来相关的问题之间有着一些形式上的相似性。这个想法在介绍层面进行了探讨，同时也指出了本文所描述的历史渊源。

同构可以通过暴力枚举的方式来决定，这是众所周知的。作为引入算法原始部分的第一步，本文的第 2 节描述了一种实际上是深度优先树搜索算法的暴力枚举过程。第 3 节介绍了工作的原始部分，它包含在树搜索中每个结点之后输入的过程。该过程的结果通常是减少必须搜索的后继结点的数量，从而减少了用来确定同构所需的总计算机用时。

在 Corneil 和 Gotlieb 的算法中 [4]，用于测试同构的两个图分别进行计算，产生具有代表性的图。类似于 Berztiss 的树搜索算法 [2]，这个我们之后将会讨论，我们的算法与 Corneil 和 Gotlieb 所提出的算法的不同在于它不会分别处理两个图：源于计算这一事实，处理子图同构的能力总是一次作用在两个子图上。我们并没有企图找到与给定图同构的所有图所拥有的图属性。

在本文中，我们将使用 Nllson [7] 的术语进行树搜索计算，以及使用 Haraty [5] 的术语表示除搜索树以外的图。图 G 由被称为点的 p 个元素的有限非空集合 V 以及属于 V 的 q 个不同的无序点对的集合 E 组成。属于 E 的一对点为一行。G 的子图是所有点和线都属于 G 的图。图 $Gα$ 与图 $Gβ$ 子图同构， 前提是该子图的点集和保持邻接的 $G_α$ 的点集之间存在 1 : 1 对应关系。

2. 用于子图同构的简单迭代算法

本节我们制定了一个简单的树搜索算法，出于介绍的目的，我们省略了在搜索中消除后继结点的关键过程。该过程会在第三节中介绍。

该迭代算法旨在找到给定图 $Gα=(Vα, Eα)$ 与另一个给定图 $Gβ=(Vβ, Eβ)$ 的子图之间的所有的同构。$Gα$ 和 $Gβ$ 的点和线的数量分别为 $pα$ ，$qα$ 和 $pβ$，$qβ$ 。$Gα$ 和 $Gβ$ 的邻接矩阵分别为 $A=[a{ij}]$ 和 $B=[b{ij}]$ 。

原因很快就会显现出来，我们将 $M’$ 矩阵定义为 $pα(rows) \times pβ(columns)$ 的矩阵，其中的元素都是 1 和 0，每行只包含一个 1，没有一列包含多个 1。矩阵 $M’=[m{ij}’]$ 可用于置换 $B$ 的行和列进一步产生矩阵 $C$。特别地，我们定义 $C=[c{ij}]=M’(M’B)^T$ ，其中 $T$ 表示转置。若下式为真

$(\forall i \forall j){ (1 \leq i \leq p\alpha \ \ 1 \leq j \leq p\beta) } (a{ij} = 1) \Rightarrow (c_{ij} = 1)$ ( 1 )

那么 $M’$ 指定了 $Gα$ 和 $Gβ$ 的子图之间的一个同构。在这种情况下，如果 $m{ij}’=1$ ，那么在本次同构中， $G\beta$ 中的第 $j$ 点对应与 $G_\alpha$ 中的第 $i$ 点。

在枚举算法的开始， 我们根据以下规则构造一个有 $p\alpha \times p\beta $ 个元素的矩阵 $M^0 = [m_{ij}^0]$

$m{ij}^0=\begin{cases} =1&{若 G\beta 的第 j 个点的度大于等于 G_\alpha 第 i 个点的度}, \ =0&{否则} \end{cases}$

事实上，在任何子图同构中，如果我们有先验理由来确定 $G\beta$ 的第 $j$ 点不与 $G\alpha$ 的第 i 点对应，那么我们就让 $m[i][j] = 0$ 。

枚举算法的工作原理是生成所有可能的矩阵 $M’$ ，使得对于 $M’$ 的每一个元素 $m{ij}’$ ，存在 $(m{ji’} = 1) \Rightarrow (m{ji}^0 = 1)$ 。对每一个这样的矩阵 $M’$ ，算法通过应用条件（1）来测试同构。矩阵 $M’9$ 通过系统地把矩阵 $M’$ 中每行的元素更改为只有一个 1 而其他元素为 0 来生成的，但必须符合矩阵 $M’$ 中每列不能包含多个 1 这一明确条件。在搜索树中，所有终端结点处于深度 $d = p\alpha$ 并且它们对应于不同的矩阵 $M’$ 。处于深度 $d < p_\alpha$ 的每个非终端结点都对应于不同的矩阵 $M$ ，与矩阵 $M^0$ 不同的地方在于在 d 行中除了一个 1 之外，其他所有元素都被更改为 0。

算法使用了一个 $p\alpha$ 位长的二元向量 $\left{F{1}, \cdots, F{i}, \cdots, F{p{\beta}}\right}$ 来记录在计算的中间状态下哪一列已经使用过了。如果第 $i$ 列已经被使用过了，则 $F{i} = 1$ 。算法还使用了一个向量 $\left{H{1}, \cdots, H{d}, \cdots, H{p{\alpha}} \right}$ 来记录选择的列的深度。如果在深度 $d$ 下选择了第 $k$ 列，则 $H{d} = k$ 。

我们将使用符合 $:=$ 来表示赋值。因此 $d := d + 1$ 意味着 “ 让 $d$ 等于 $d + 1$ ” 。此外，我们写 $M := M_d$ 的意思就是 “ 让整个矩阵 $M$ 等于矩阵 $M_d$ ” 。矩阵 $M_d$ 是矩阵 $M$ 在深度为 $d$ 时的存储副本。我们已经制定了算法，使其尽可能与第三节中的算法相似（比如，完全赋值 $M_d := M$ 在本节中并不是必需的）。

简单迭代算法如下所示:

第 1 步 $M = M^0, d := 1; H1 = 1;\ for \ all\ i = 1, \cdots, p\alpha, set \ F_i = 0$

第 2 步 $如果不存在值\ j\ 使得\ m_{dj}=1\ 以及\ F_i = 0，那么执行第\ 7\ 步; \ M_d := M, \ if\ d = 1\ then\ k := H_1 else k := 0,$

第 3 步 $k := k + 1, \ 如果\ m_{dk} = 0\ 或者\ Fk = 1\ 那么执行第\ 3\ 步;\for\ all\ j \ne k\ set\ m{dj} := 0$

第 4 步 $如果\ d < p_\alpha\ 则执行第\ 6\ 步，否则使用条件（1）并且如果找到同构则输出$

第 5 步 $如果不存在\ j>k\ 使得\ m_{dj} = 1\ 且\ F_j = 0\ 那么执行第 7 步;\ M := M_d,\ 执行第\ 3\ 步;$

第 6 步 $H_d = k, F_k = 1; d := d + 1;\ 执行第\ 2\ 步,$

第 7 步 $如果\ d=1 那么终止算法,\ F_k = 0; d := d - 1, M := M_d, k := H_d,\执行第\ 5\ 步,$
