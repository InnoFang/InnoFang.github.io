是时候用 Github 私仓做你的私人图床了
2019-01-08|Github

喜大普奔！Github 终于开放了对普通用户的私人仓库的使用 🎉。

自从 Github 被微软收购以后，可谓动作频频，无论是有 UI 还是其他方面都有了很多的变化，不过在 1 月 7 日这天宣布开放私人仓库的使用，估计有不少人都会说：真香！。

Github 对每个代码仓库的大小限制大概是 1G ，虽然这个 1G 不算很大，但是用来做自己博客文章的图床我觉得还是可以的，毕竟有就不错了，要啥自行车呢 （手动滑稽）

我其实尝试过很多图床，但最后都因为各种各样的原因不再继续使用了，最后是在自己的笔记仓库上建了一个分支，用来做图床，这样相对不容易被发现，毕竟我个人感觉在这种开源的环境建一个图床仓库还是有点尴尬的，但是私仓的出现就可以避免这种尴尬了。

建一个私仓吧

要注意勾选 Private。不过如果忘了的话，也可以到新建的项目中，通过 settings -> Danger Zone -> Make this repository private 把你的仓库变成私仓。

Git 三连

把空的仓库 clone 到本地后，把你的图片放到项目对应的本地文件夹中，接下来就是传统的 Git 三连了

属于你的 Github 私人图床就 OK 了！

不过，如果只是这样的话，那这篇文章就没啥价值了，因为现在这个私人仓库只有你能访问到，所以如果是直接通过 右键 -> 复制图片地址 的方式来引用图片的话，将出现 404。😮

换一个思路

拿我图床中的一个图片为例子，通过 右键 -> 复制图片地址 获得的图片链接是这样的

通过这个连接可以发现，其实这相当于是从外部访问你的私仓，这当然是访问不到的

那么怎么解决呢？其实如果 右键 -> 在新标签页中打开图片 ，你会发现你原来的图片链接由被重定到了新的链接

其中开头的 https://raw.githubusercontent.com 是 Github 的转存地址，如果你点开该链接会被重定向到 Github 主页（https://github.com），据说这是 Github 的素材服务器 (assets server)，是为了避免跟主服务抢占负载。
(PS：更多解释可以查看 What do raw.githubusercontent.com URLs represent?)

总而言之，这个链接外界是可以访问到的，那么我们的问题就引刃而解了。

多说一句

如果小伙伴之前是用的公有仓库作图床的话，可以自行选择是否把仓库变为私仓，修改后，难免就需要修改之前不合适的链接；如果小伙伴之前使用的是其他图床，且正好需要一个新图床，那么可能也需要面临修改图片链接的问题。

大部分情况下，我们在 MarkDown 文件中引用图片资源是通过 ![desc](url) 的方式来引用的，但是如果你尝试对大篇幅的文章更换图片链接的话，会发现这是一个很痛苦的过程，那么既然要更改图片链接，是否有一个相对易于修改的方式呢？
